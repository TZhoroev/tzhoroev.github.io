Uncertainty Quantification for Physical & Biological Models

NC State University · National Science Foundation

🔬 NSF-Funded Research at North Carolina State University

Every mathematical model of a physical or biological system is, at best, an approximation. The real question isn't "what does the model predict?" — it's "how much should we trust that prediction, and where does the model break down?" This NSF-funded research developed Bayesian methods for answering both questions: propagating uncertainty through model parameters to build reliable prediction intervals, and detecting when the gap between model and data signals a structural problem rather than just noise.

Six Project Areas

Sensitivity Analysis

Before spending compute on full calibration, figure out which inputs actually matter. Local and global sensitivity methods rank parameters by influence, focusing effort where it counts.

Parameter Estimation

The classical starting point: find the parameter values that make the model best match observed data, using well-defined objective functions and optimization.

MCMC Sampling

When a single best-fit isn't enough, we need the full posterior distribution. Markov chain Monte Carlo methods explore the parameter space to map out what's plausible — and how plausible.

Frequentist Intervals

Sometimes you don't need or want a full Bayesian posterior. Confidence and prediction intervals provide quick, interpretable uncertainty bands using classical statistical theory.

Surrogate Modeling

When the real simulator takes hours to run, build a fast approximation that preserves the important input-output relationships. Essential for making MCMC and sensitivity analysis tractable.

Model Discrepancy Detection

The hardest question: is the mismatch between model and data caused by uncertain parameters, or is the model itself structurally wrong? These diagnostics help tell the difference.

The Bayesian foundation

Everything starts from a model \(y = \mathcal{M}(\theta) + \varepsilon\) and some prior belief about the parameters. Bayes' theorem turns the problem of "what parameters are plausible?" into a clean mathematical question:

\[ p(\theta \mid y) = \frac{p(y \mid \theta)\,p(\theta)}{p(y)} \]

The posterior distribution \(p(\theta \mid y)\) tells us everything we need: the most likely parameter values, how uncertain we are about them, and how that uncertainty propagates into predictions.

How MCMC explores the posterior

For most real problems, we can't compute the posterior analytically. Instead, we draw samples from it using Metropolis-Hastings, accepting or rejecting proposals based on:

\[ \alpha(\theta, \theta') = \min\!\left(1, \frac{p(y\mid\theta')p(\theta')q(\theta\mid\theta')}{p(y\mid\theta)p(\theta)q(\theta'\mid\theta)}\right). \]

With enough samples, we can estimate prediction intervals, identify multi-modal posteriors, and quantify exactly how much the data constrains each parameter.

From screening to inference

The portfolio is structured so that sensitivity analysis results directly inform the calibration — screened-out parameters get fixed, focusing MCMC on the dimensions that matter.

Beyond fitting
